# Réalisation Par 监控助手（中文说明）

[English version](./README.md)

一个跑在 Google 账号上的后台小程序，专门盯着一个服装品牌官网和你的 Gmail 收件箱，监控两件平时很容易错过的事：全场大促的邮件通知（经常被 Gmail 自动分到"促销"标签页里吃灰）、以及下架款/卖光款的重新上架。完全跑在 Google 自己的服务器上，不需要额外的服务器，不依赖第三方付费服务，免费。

## 这个项目能做什么

- **大促邮件检测** —— 扫描 Gmail 里来自该品牌的促销邮件，不管 Gmail 把它分到哪个标签页都能发现。
- **补货检测** —— 监控具体款式的两种状态：完全下架（连页面都没了）、以及"last sale"全码卖光（页面还在，只是没货）。可以精确到只关心某一个具体尺码。
- **降价检测** —— 即使不是全场大促，单品自己悄悄降价也能被发现，并把价格变化记录下来方便后续画图。
- **汇总通知** —— 一次检查发现的所有变化会合并成一封邮件（如果有的话还会带商品图片），而不是一件件分开轮番轰炸收件箱。

## 这是什么 / 工作原理

这是一个运行在 Google 账号上的小程序，通过一个叫 **Apps Script** 的免费工具运行——不需要在自己电脑上装任何东西，电脑关机、不开网页都没关系，它会一直在 Google 的服务器上自己跑。

跑起来之后，它每30分钟会做这几件事：
1. 检查 Gmail 里有没有品牌发来的促销邮件
2. 检查官网上你关心的那些款式有没有动静
3. 把发现的内容，从你自己的 Gmail 账号发邮件给你自己

用到两个免费的 Google 工具：
- **Google Sheets** —— 一张表格，用来列出你要监控的款式，结果也会写在这里
- **Google Apps Script** —— 负责定时跑检查逻辑，并发提醒邮件

不需要额外装App，不花钱，只需要粘贴一段代码进去。

提醒邮件是从你自己的 Gmail 账号发给你自己的普通邮件。因为是"自己发给自己"这种性质，不是群发的营销邮件，Gmail 的自动分类几乎不会把它分进"促销"标签页（不像品牌自己发的促销邮件那样）。

### 款式"消失"的两种情况，这点很关键

品牌官网通常有两种不同的"下架"方式，表格里要分别对应填法：

- **完全下架** —— 这款衣服彻底消失，连页面链接都没了。只需要填 **Keyword** 这一列（填款式名字就行）。脚本会去扫商品列表页，看这个名字有没有重新出现。
- **"last sale" / 全码卖光** —— 商品页面还在、能打开，只是所有尺码都显示没货。除了 Keyword，还要把商品页面的实际链接填进 **Product URL** 这一列。脚本会直接去查那个页面里"是否所有尺码都没货"这个真实状态，这比单纯扫列表页准得多。
- **只关心某一个具体尺码** —— 在填了 Product URL 的基础上，再填上 **Size** 这一列（比如填 `M`、`XL`）。这样只有你要的那个尺码恢复有货时才会提醒，不会因为别的尺码先来货就误报。

下面建表格的步骤里会用到这三列。

### 降价提醒

除了补货，只要某一行填了 Product URL，脚本每次检查时也会顺便记一下价格。即使不是全场大促，单品自己悄悄降价也能被发现——脚本会记住上次看到的价格（存在 **LastPrice** 这一列），只有当新价格比上次更低时，才会在提醒邮件里提到。这个功能只对填了 Product URL 的行有效；完全下架、只填了 Keyword 的行没有现成页面可以读价格。

### 一次检查只发一封邮件，不是一件件分开发

如果一次检查同时发现好几件事（比如两件补货 + 一次降价），会**合并成一封邮件**，列出所有变化，而不是连续收到好几封邮件轰炸你。如果什么都没变化，这次检查就不会发任何邮件。对于填了 Product URL 的行，邮件里还会带一张商品图片（直接从商品页面读到的）；完全下架、只填 Keyword 的行因为没有页面可读，不会带图。

### 把价格历史画成图表

每次监控的商品价格发生变化，`PriceHistory` 这张表里就会自动多一行记录（日期、款式名、价格）。攒了几条数据之后：

1. 选中这些数据（包括表头那一行）
2. 点菜单的 **插入 → 图表**
3. 图表类型选 **折线图**

这个图表会随着新数据自动更新，不用每次重新做。如果同一张表里监控了好几款不同的衣服、想给每一款单独画一张图，可以先用 **数据 → 创建筛选条件视图** 筛一下，再插入图表。

## 第一步 —— 建表格

1. 浏览器打开 **sheets.new**，会自动新建一个空白表格。
2. 左上角点"未命名的电子表格"，改个名字，比如 `RP监控`。
3. 屏幕最下面有个写着 `Sheet1` 的分栏，右键它，选**重命名**，改成完全一致的：`Watchlist`
4. 在 `Watchlist` 这个分栏的第一行，依次填这七个表头，一格一个：

   | A1 | B1 | C1 | D1 | E1 | F1 | G1 |
   |---|---|---|---|---|---|---|
   | Keyword | Product URL | Size | Notes | Status | LastChecked | LastPrice |

5. 从第二行开始，按照前面说的三种情况，把你要监控的款式一行行填进去。

   - **完全下架的款**，只填 A 列：

     | A（Keyword） | B（Product URL） | C（Size） | D（Notes） |
     |---|---|---|---|
     | Mathilde | | | 之前下架的那条裙子 |

   - **链接还在但全码卖光的款**，填 A 和 B 两列，C 列留空表示"任意尺码恢复都提醒"：

     | A（Keyword） | B（Product URL） | C（Size） | D（Notes） |
     |---|---|---|---|
     | Gia Havana | https://int.realisationpar.com/the-gia-havana/ | | 标记了last sale |

   - **只想要某个具体尺码**，再多填 C 列：

     | A（Keyword） | B（Product URL） | C（Size） | D（Notes） |
     |---|---|---|---|
     | Meg Rosette | https://int.realisationpar.com/the-meg-rosette/ | XL | 只想要XL |

   E、F、G 三列留空，脚本会自动填（Status 会写成 `In Stock` 有货 / `Out of Stock` 缺货 / `Not Found` 找不到；LastChecked 会写时间；LastPrice 只对填了 Product URL 的行有效）。

6. 点左下角的 **+** 号新建一个分栏，重命名成完全一致的：`SaleLog`
7. `SaleLog` 第一行随便填表头，比如 `日期`、`标题`、`摘要`——这张表会自动往里加内容，不用你手动填。
8. 同样的方法再加一个分栏，重命名成完全一致的：`PriceHistory`
9. `PriceHistory` 第一行填三个表头：`日期`、`款式`、`价格`——同样会自动往里加内容。
10. 看一下浏览器地址栏，网址长这样：

    ```
    https://docs.google.com/spreadsheets/d/1AbCxyz123这一串就是ID/edit
    ```

    把 `/d/` 和 `/edit` 中间那一长串字符复制下来，这就是你的**表格ID**，存好，下一步要用。

---

## 第二步 —— 建 Apps Script 项目

1. 浏览器打开 **script.new**，会自动新建一个空白项目。
2. 里面有个文件叫 `Code.gs`，默认有一些内容（可能就是一行 `function myFunction() {}`），全部删掉。
3. 把仓库里的 `Code.gs` 整段内容复制粘贴进这个空文件里。
4. 找到代码靠前面的这一行：

   ```
   const SHEET_ID = 'PASTE YOUR GOOGLE SHEET ID HERE';
   ```

   把引号里的占位文字换成你刚才复制的表格 ID，引号不要删，改完应该长这样：

   ```
   const SHEET_ID = '1AbCxyz123example';
   ```

5. 左上角点项目名字（"无标题项目"），改个名字。
6. 按 **Ctrl+S**（Mac 是 **Cmd+S**）保存。

---

## 第三步 —— 授权读取和发送 Gmail 的权限

这一步看起来吓人，其实没事——这是 Google 对所有个人脚本的标准提示，因为这种脚本没有走 Google 的公开应用审核（那个审核只针对面向大众的 App，不是只有自己用的私人脚本）。

1. 屏幕上方有个函数下拉框和一个 **Run** 按钮（▶ 三角形播放图标）。
2. 下拉框选 `testNotification`。
3. 点 **Run**。
4. 弹出"需要授权"的窗口，点**查看权限**。
5. 选你的 Google 账号。
6. 大概会看到一个"Google未验证此应用"的提示，点**高级**（小字链接），再点**转至xxx(不安全)**。

   这个"不安全"提示是正常现象，只是说明这个脚本没经过 Google 的公开审核，因为只有你自己用。
7. 点**允许**。
8. 去 Gmail 收件箱看一眼，应该几秒内会收到一封标题是 `Test notification` 的邮件。

如果没收到，先看一眼垃圾邮件箱，再去 Apps Script 页面下方的 **Execution log** 看看有没有报错信息。

---

## 第四步 —— 开启自动定时检查

1. 函数下拉框选 `setup`。
2. 点 **Run**。
3. 这会建好两个定时任务，之后每30分钟自动运行一次——一个检查邮件，一个检查官网。
4. 可以点左侧菜单的时钟图标（**Triggers**）确认一下，应该能看到两条记录。
5. 同时也会收到一封标题 `Setup complete` 的邮件。

做完这一步就结束了。之后它会一直在后台自己运行，不需要再开着这个网页标签页。

## 日常怎么用

- **加新的监控款式：** 直接去表格 `Watchlist` 分栏加一行就行，不用重新跑任何代码。
- **看促销邮件的历史记录：** 去 `SaleLog` 分栏看，会自动累积。
- **改检查频率：** 回 Apps Script 项目，点左侧时钟图标（**Triggers**），在那里调整。
- **只想监控裙子（缩小扫描范围）：** 去 `Code.gs` 里找 `CATALOG_PAGES_TO_SCAN` 这一行，把里面的网址换成裙子分类的网址。

## 常见问题排查

- **完全没收到邮件：** 先看垃圾邮件箱，再去 Apps Script 的 **Execution log** 看 `testNotification` 或 `setup` 有没有报错。
- **报错提示"找不到分栏"：** 检查三个分栏名字是不是完全对得上 `Watchlist`、`SaleLog`、`PriceHistory`（大小写也要对）。如果只是 `PriceHistory` 没建好，只会影响价格历史记录，不会影响补货和降价检测本身。
- **某封促销邮件没有触发提醒：** 去 `Code.gs` 找 `SALE_KEYWORDS` 这个列表，把那封邮件标题里的关键词加进去。
- **某一行 Product URL 一直显示 "Not Found"：** 先确认链接复制完整、能不能正常打开。如果链接没问题但还是不对，可能是网站改版了判断库存的方式。
- **表格里有些行的 Status/LastChecked 一直是空的，没被处理：** 通常是某一次运行因为某一行的网页加载太慢，超过了 Apps Script 单次运行的时间上限被中断了。因为每次运行都是从头开始处理整张表，下一次定时运行通常会自动补上没处理完的行。如果总是卡在同一行，可以自己打开那个链接确认是不是真的能正常加载。

## 已知局限

- 库存/尺码/价格的解析依赖目标网站现在的网页结构（具体来说是一个叫 `BCData` 的数据块和几个 `og:` 标签）——如果品牌官网改版，可能需要更新解析逻辑。
- 单次运行偶尔会因为某一页加载太慢，超过 Apps Script 的单次执行时间上限被中断；下一次定时运行会从头重新检查一遍，自动纠正。
- `SHEET_ID` 目前是写在 `Code.gs` 里的一个普通变量，方便上手；如果以后要分享给别人用或者公开仓库挂真实ID，更安全的做法是改成从 Apps Script 的 Script Properties 里读取。

## 技术栈

Google Apps Script · Gmail API · Google Sheets API · UrlFetchApp（没有用任何外部HTTP库，也没有调用任何大模型/AI接口）
